# File Lancet 项目需求说明书

## 1. 项目概述

### 1.1 项目背景

File Lancet 是一款面向开发者和高级用户的文件结构分析工具。项目的初衷是解决 EPUB 电子书阅读应用中，对文件结构解析准确度难以评估的问题。它旨在像"手术刀"一样，精准地解剖文件，展示其内部的物理结构和逻辑内容。

### 1.2 核心目标

- **精准解析**：深度解析 EPUB 文件，还原其目录结构、元数据、阅读顺序及具体内容。
- **可视化分析**：通过直观的三栏式界面，展示文件层级、属性详情和内容预览。
- **高扩展性**：采用插件化或策略模式架构，未来能够轻松支持其他文件格式的解析。

## 2. 功能需求

### 2.1 文件输入与加载

#### 多方式打开

- 支持拖拽 `.epub` 文件至应用窗口进行加载。
- 支持通过文件对话框选择文件。
- 支持操作系统层面的文件关联（右键"打开方式"）。

#### 加载反馈

- 解析过程中需提供加载状态提示，解析失败（如文件损坏）需报错。

### 2.2 核心解析功能 (EPUB)

系统需能够识别并提取以下信息：

#### 物理结构

- 识别 ZIP 容器内的所有文件及文件夹层级。

#### 逻辑结构

- **元数据**：提取书名、作者、出版社、语言、ISBN 等。
- **格式版本**：识别 EPUB 版本（如 2.0, 3.0, 3.1）。
- **阅读顺序**：解析 Spine 信息，确定章节排列顺序。
- **目录结构**：解析 NCX 或 NAV 文档，提取层级化目录。

#### 内容提取

- 能够读取 HTML/XHTML 正文源码、CSS 样式表及二进制资源（图片/字体）。

### 2.3 界面交互需求 (双栏式布局)

界面采用双栏布局，左侧为文件树和详情面板（上下堆叠），右侧为内容预览区，支持拖拽调整宽度。

#### 左栏：文件树 + 详情面板（上下布局）

**上部 - 文件结构树：**

- 以树形控件展示文件的层级结构。
- 点击节点可展开/折叠，选中节点触发详情面板和预览区更新。
- 显示文件名和节点类型。

**下部 - 详情面板：**

根据选中的节点类型，动态显示不同信息：

- **物理属性**：名称、类型、大小、路径。
- **书籍元数据**（对根节点/配置文件）：书名、作者、出版社、语言、版本等。

#### 右栏：预览

- **HTML/XHTML**：树状结构显示 DOM 层级。
- **CSS/JS/XML**：代码高亮显示，支持结构化树状视图。
- **图片**：直接显示图像预览。
- **二进制文件**：十六进制格式显示，每行 32 字节，包含偏移量、十六进制值和 ASCII 字符。

#### 底部状态栏

- **通用信息**：始终显示当前打开文件的完整路径、总大小、最后修改日期。
- **上下文提示**：当鼠标悬停或选中左/中栏某项时，显示该元素的简短描述（如"这是定义书籍阅读顺序的配置节点"）。

## 3. 非功能需求

### 3.1 架构设计

- **高内聚，低耦合**：核心解析逻辑必须与 UI 层完全分离。
- **扩展性设计**：
  - 定义通用的 `IFileParser` 接口。
  - 解析器需独立实现，确保未来增加 PDF 或 MOBI 支持时，无需重构核心框架。
- **数据标准化**：解析结果需转换为统一的标准数据模型（如 `FileNode` 树），供 UI 层绑定。

### 3.2 性能

- **内存管理**：对于大文件（如包含大量高清图片的 EPUB），需采用懒加载策略，仅在用户点击查看时读取文件流，避免一次性占用过多内存。
- **响应速度**：UI 线程不得阻塞，文件解析需在后台线程异步执行。

### 3.3 兼容性

- **Windows**：完美支持 Windows 10/11（x64、ARM64）。

## 4. 技术路线选型

基于项目需求（工具类、强逻辑、Windows 桌面应用），经过对比分析，确定以下技术栈：

| 模块     | 技术选型                                  | 选型理由                                                                       |
| ------ | ------------------------------------- | -------------------------------------------------------------------------- |
| 开发语言   | C# (.NET 8/9)                         | 拥有强大的原生文件处理能力 (`System.IO`, `System.Xml`)，强类型系统适合构建严谨的数据模型，且 .NET 跨平台性能优异。 |
| UI 框架  | WPF (Windows Presentation Foundation) | Windows 原生 UI 框架，与 Windows 系统深度集成，提供丰富的控件库和成熟的数据绑定机制，适合开发高性能桌面工具。          |
| 架构模式   | MVVM                                  | 推荐采用 Model-View-ViewModel 模式，利用数据绑定机制，将解析逻辑与界面展示解耦。                        |
| 核心库    | `System.IO.Compression`               | 用于解包 EPUB (ZIP) 容器。                                                        |
| <br /> | `System.Xml.Linq`                     | 用于解析 OPF, NCX, Container.xml 等配置文件。                                        |
| <br /> | HtmlAgilityPack (可选)                  | 用于解析和遍历 HTML 文档结构。                                                         |
