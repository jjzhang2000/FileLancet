## v0.1.0 需求（2026-03-26）

### 1. 项目概述

File Lancet 是一款面向开发者和高级用户的文件结构分析工具，旨在像"手术刀"一样精准解剖文件，展示其内部物理结构和逻辑内容。

### 2. 功能需求

#### 2.1 文件输入与加载
- 支持拖拽 `.epub` 文件至应用窗口进行加载
- 支持通过文件对话框选择文件
- 支持操作系统层面的文件关联
- 解析过程中需提供加载状态提示

#### 2.2 核心解析功能 (EPUB)
- 识别 ZIP 容器内的所有文件及文件夹层级
- 提取元数据（书名、作者、出版社、语言、ISBN等）
- 识别 EPUB 版本（2.0, 3.0, 3.1）
- 解析 Spine 信息，确定章节排列顺序
- 解析 NCX 或 NAV 文档，提取层级化目录
- 读取 HTML/XHTML 正文源码、CSS 样式表及二进制资源

#### 2.3 界面交互需求
- 双栏式布局：左侧文件树+详情面板，右侧内容预览区
- 文件树以树形控件展示层级结构
- 详情面板动态显示节点信息（物理属性、书籍元数据）
- 预览支持：HTML/XHTML、CSS/JS/XML、图片、十六进制
- 底部状态栏显示文件路径、大小、修改日期

### 3. 非功能需求
- 核心解析逻辑与 UI 层完全分离
- 定义通用的 `IFileParser` 接口，支持扩展
- 采用懒加载策略，优化内存占用
- UI 线程不得阻塞，文件解析异步执行
- 支持 Windows 10/11（x64、ARM64）

### 4. 技术选型
- 开发语言：C# (.NET 8/9)
- UI 框架：WPF
- 架构模式：MVVM
- 核心库：`System.IO.Compression`、`System.Xml.Linq`

## v0.1.1 需求（2026-03-29）

### 1. 版本概述

v0.1.1 是维护版本，主要增加通用文件支持功能，使应用能够打开任何格式的文件，不再局限于 EPUB 格式。

### 2. 新增功能需求

#### 2.1 通用文件支持（兜底功能）
| 需求编号 | 需求描述 | 优先级 | 状态 |
|---------|---------|--------|------|
| REQ-701 | 实现 GenericFileParser 通用文件解析器 | 高 | ✅ 已完成 |
| REQ-702 | 支持打开任何格式的文件 | 高 | ✅ 已完成 |
| REQ-703 | 不支持的格式默认以十六进制方式预览 | 高 | ✅ 已完成 |
| REQ-704 | 文件树只保留文件名（无子节点结构） | 中 | ✅ 已完成 |
| REQ-705 | 详情面板显示文件名、路径、大小、修改日期 | 高 | ✅ 已完成 |
| REQ-706 | 详情面板显示扩展名、MIME 类型 | 中 | ✅ 已完成 |
| REQ-707 | 更新 ParserFactory，通用解析器最后注册作为兜底 | 高 | ✅ 已完成 |

#### 2.2 数据模型扩展
| 属性名 | 类型 | 说明 | 状态 |
|-------|------|------|------|
| CreatedTime | DateTime | 文件创建时间 | ✅ 已完成 |
| FileExtension | string | 文件扩展名 | ✅ 已完成 |
| MimeType | string | MIME 类型 | ✅ 已完成 |
